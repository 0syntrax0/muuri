About

What does it do?

Essentially Muuri is a combination of Packery, Isotope and jQuery UI sortable. It can position items in a grid based on the default algorithm or a custom algorithm, make the items draggable and hide/show/add/remove items. There are no built-in methods for filtering and sorting the grid items because it's trivial to implement your own sorting and filtering functinality with the API.

What's up with the name Muuri?

Muuri is finnish and means "wall".

Can I use it in production?

Well there are no unit tests yet and the API is likely to change until the magical v1.0.0 is reached. There are also some big features missing (like dragging an item from a muuri instance to another) which are planned for v1.0.0. If you're fine with all of that then go ahead and use it.

Features

  • 

    Customizable layout

    By default Muuri uses "First Fit" bin packing algorithm to position the items in the grid. The default layout mode also has a few options which allows you to define the vertical (top/bottom) and horizontal (left/top) alignment of the items and also the orientation of the whole layout (portrait/landscape). The portrait mode works for vertically scrolling sites and the landscape mode works for horizontally scrolling sites. You can also provide your own layout algorithm.
  • 

    Fast animations

    Fast and highly customizable positioning and show/hide animations powered by Velocity.js.

  • 

    Drag & drop

    Built-in support for dragging and dropping the grid items powered by Hammer.js. The drag and drop stuff is completely optional and Hammer.js is not required as a dependency if you are not using the drag and drop feature.

  • 

    Rich API

    Rich API with enough methods and events to gain full control over the library's behaviour. For example custom filtering and sorting of the grid items can be implemented easily (check the source of this site for starters).

  • 

    Good browser support

    Works in all modern browsers and IE9+.

  • 

    Fully open source
